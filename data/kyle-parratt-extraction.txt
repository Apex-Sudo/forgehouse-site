Case Study: AI Agent Orchestration Platform 🤖
Role:
Lead Applied AI Architect
Objective:
Transform a complex CRM/Gamification system into a "Conversational Operating System"
where back-office tasks are handled by autonomous AI agents.
The Challenge:
Standard CRM interfaces require high cognitive load and manual configuration. The goal
was to build a system where users could describe complex outcomes (e.g., "Build a
high-converting popup for a weekend promo") and have AI agents execute the
end-to-end workflow, including asset generation, data querying, and UI assembly.
The Solution:
A Modular Multi-Agent Framework. I architected and built a production-grade AI platform
from the ground up, utilizing LangGraph for state-machine orchestration and Next.js
15/React 19 for a real-time streaming interface.
Key Architectural Pillars:
● Agentic State Machines LangGraph): Moved beyond simple prompting to
directed acyclic graph DAG workflows. Each agent Popup, Analytics,
Gamification) operates as a state machine with dedicated planning, execution, and
validation nodes.
● Handler Registry Pattern: Developed a generic AgentHandlerRegistry allowing for
modular "plug-and-play" agent deployment. This architecture separates generic
infrastructure (streaming, auth) from agent-specific domain logic.
● Autonomous Analytics Text-to-SQL: Engineered a Transactions Agent that
autonomously explores BigQuery schemas, generates optimized SQL, and
visualizes results via Vega-Lite, enabling non-technical users to perform complex
data science tasks.
● Multi-Modal Asset Creation: Integrated Google Imagen API within agent
workflows to generate game assets (icons, backgrounds, badges) dynamically
based on the specific context of the userʼs gamification strategy.
Technical Excellence & Innovation:
● Real-time Observability: Implemented SSE Server-Sent Events) streaming with
"Thinking" transparency, showing the AIʼs step-by-step reasoning and progress
checklists to increase user trust and system reliability.
● Token Lifecycle Management: Developed a granular tracking system for input,
output, and "thinking" tokens, including per-label quotas and usage tracking to
ensure cost-efficiency across 30 brands.
● Feedback Loops: Integrated a "Message Rating" system and draft-loading
mechanics, allowing for iterative refinement of AI-generated content.
The Tech Stack:
● AI/Backend: LangGraph, LangChain, Google Gemini, PostgreSQL, BigQuery.
● Frontend: Next.js 15, React 19, @assistant-ui/react, Zustand, Tailwind 4.
● Infrastructure: AWS S3/CloudFront/ALB, SSE Streaming, JWT Jose.
Real Life Impact:
● Reduced Time-to-Live: Cut the time to create complex CRM popups and
gamification skins from hours of manual work to under 60 seconds of
conversation.
● Democratized Data: Removed the SQL bottleneck by allowing back-office staff to
query 1M row datasets through natural language.

# Kyle Parratt: AI Systems Architect

## Core Value Proposition
Kyle is the engineer companies call when they need production-grade AI systems that actually work, not demos. He's a "10x engineer" who can rapidly build fault-tolerant AI solutions from 0 to 1, specializing in taking experimental AI capabilities and turning them into deterministic, enterprise-ready software.

## Diagnostic Framework

### Initial Client Assessment
**Key Questions:**
- "What's your biggest pain point?"
- "What does the perfect solution look like to you?"
- "Can you go into more detail?"
- "How many users? Expected response time? Budget?"

**Red Flags (Project Will Fail):**
- Blurry, poorly defined specs
- Vanity projects just to say they use AI
- Unrealistic expectations (like $100K chatbot for stock predictions)
- Client can't articulate clear metrics, milestones, or deliverables
- "Dump everything into LLM and get magic out" mentality

**Green Flags (Project Will Succeed):**
- Clear pain point solving real business problem
- Well-informed client with specific requirements
- Realistic budget and timeline expectations
- Understanding that AI is a tool, not magic

### Technical Viability Assessment
**Core Questions:**
- Does this solve a real business use case?
- Is there an existing tool that already does this?
- How long for MVP development?
- Does it fit within financial and resource constraints?

## Core Methodology

### Architecture Philosophy
1. **Start Simple:** "Less is more" - build MVP first, add complexity later
2. **Modular Design:** Create reusable base classes, clear separation of concerns
3. **Handle Unhappy Paths:** Everything can break and should be handled gracefully
4. **Extensive Observability:** "YOU WILL have bugs and logs make these easier to debug"

### The "GIGO" Principle
- LLMs are "just inference engines" - they're stupid
- Real value is in the data pipeline feeding the agent
- Good data can make even a bad model look good
- No amount of fancy loops or agentic patterns can overcome bad data

### Data Quality Framework
**Data Hierarchy:**
1. Some data > no data
2. Relevant and helpful > irrelevant
3. Good signal-to-noise ratio
4. Comprehensive coverage
5. Dense > sparse
6. Validated and reliable
7. Change tracking implemented

### Production Readiness Checklist
- Centralized, modular, reusable code
- Base classes for common functionality
- Clear folder structures and separation of concerns
- Error handling for both happy and sad paths
- Retries and fault-tolerant orchestration (like Temporal)
- Security and guardrails against prompt injection
- Systematic benchmarking and monitoring

### Build vs. Buy Decision Framework
**Always Buy:** Auth, API wrappers, PDF libraries, loggers - use battle-tested solutions
**Build Yourself:** Business logic and domain-specific functionality
**Cost Calculation:** Week to build + week to debug vs. using existing solution means engineers aren't working on actual business problems

## Quality Assurance Approach

### Code Review Red Flags
- Tons of changes for simple requests (AI overcomplicated)
- Convoluted solutions when simpler approaches exist
- Duplicated/scattered code
- Functions polling APIs every few seconds
- Secrets in frontend
- Terrible folder structure

### AI Code Management
1. Carefully review all AI-generated code
2. Question AI's approaches - it takes easiest solution, not most production-ready
3. Inform AI of mistakes and ask for corrections
4. Never just commit without understanding

### Benchmarking Strategy
**Metrics to Track:**
- Accuracy (using LLM-as-judge)
- Token usage
- Latency
- Cost

**Process:** Run experiments 2-3 times for averages, keep improvements, discard regressions

### Prompt Engineering Process
1. Build MVP prompt
2. Iterative refinement based on production issues
3. If prompt engineering hits limits, split workload into manageable tool calls
4. Use LangSmith or similar for systematic benchmarking

## Team vs. Solo Approach

### Solo Work
- Focus on execution
- Use consistent tech stack (Node.js, TypeScript, React, PostgreSQL)
- Rapid iteration and direct implementation

### Team Leadership
- Focus shifts to facilitation over execution
- Create modular architecture with strict interfaces
